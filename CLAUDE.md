## Project Overview

**RemoteDesk** is a browser-based remote desktop solution for Windows — a private project. All planning documents are written in German.

## Architecture

```
Windows Host (C# / .NET 10, WPF)
  DXGI Capture → VP8 (FFMpegCore) → WebSocket → Relay
  SendInput ← Input Events ← Relay ← Browser

Relay Server (ASP.NET Core, Docker on QNAP NAS)
  Session-code matching, 1 Host → N Viewer broadcast
  Self-signed SSL (cert generated by Host, deployed to QNAP)

Browser Clients (Vanilla JS, HTML5 Canvas, WebCodecs API)
  No installation required, multiple simultaneous viewers
```

## Key Design Decisions

- **Relay is a pure data forwarder** — it does not inspect content, only proxies encrypted WebSocket frames
- **Session identification** via 9-digit codes; sessions stored in-memory on relay
- **Connection model**: Host connects outbound to Relay (no port-forwarding needed on Host side); Port 443 on QNAP must be externally reachable
- **Control model**: max one viewer has remote control at a time; Host must approve control requests via WPF popup
- **Clipboard sync** uses source-IDs to prevent echo loops
- **File transfer** is chunked (256 KB) over WebSocket with transfer-IDs
- **Self-signed cert** is generated by the Host app (BouncyCastle), exported as PFX for Docker, CER for browser trust; QNAP DNS name used as CN/SAN

## WebSocket Protocol

All messages are binary or JSON with type header. Key message types: `video_frame`, `input_mouse`, `input_key`, `clipboard_sync`, `file_offer`, `file_chunk`, `session_event`, `control_request`, `control_response`, `config_sync`.

## Relay Endpoints

- `wss://relay:443/host` — Host registers here
- `wss://relay:443/viewer?code=XXXXXXXXX` — Viewer connects here
- `GET /health` — Docker health check

## QNAP-Specific Notes

- Relay runs as Docker container in Container Station
- Port 443 must be forwarded; no conflict with QNAP management (8080/8443)
- Container Station supports docker-compose

## Implementation Phases

Detailed specs are in the `RemoteDesk_Phase*.md` files and `RemoteDesk_Projektplan.md`:

1. **LAN Core** — Screen capture, VP8 encoding, WebSocket stream to browser canvas
2. **Relay & Internet** — Docker relay server, self-signed SSL, session codes
3. **Multi-Monitor & FPS** — Monitor switching, adaptive FPS 5-30, resolution scaling
4. **Remote Control** — SendInput injection, multi-viewer roles, control request/approval flow
5. **Clipboard & File Transfer** — Bidirectional clipboard sync, chunked file transfer
6. **Installer & Polish** — Inno Setup with firewall rules, full settings UI, auto-reconnect
